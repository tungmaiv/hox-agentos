<objective>
Fix the bell dropdown so it renders whenever the bell is clicked, not only when pendingCount > 0. When there are no pending skills, show an empty-state message instead of silently doing nothing.

Purpose: UAT test 5 reported "I click to the bell but nothing happened" because the dropdown was gated on `bellOpen && pendingCount > 0` (layout.tsx line 173). With 0 pending skills (the current data state), clicking the bell toggles `bellOpen` but the dropdown never renders. Adding an empty state makes the bell useful and discoverable even when the queue is empty.

Output: One-line condition change + empty-state paragraph added to the existing dropdown JSX.
</objective>
