    - revision = "016", down_revision = "015", branch_labels = None, depends_on = None

    upgrade() function body — pure SQL via op.execute(sa.text()):

    For each of the 6 tables: memory_facts, conversations, turns, credential_store, workflow_runs, workflow_run_results:
    1. ENABLE ROW LEVEL SECURITY on the table
    2. FORCE ROW LEVEL SECURITY (so table owners are also subject to RLS — defense-in-depth)
    3. CREATE POLICY "user_isolation" ON {table} USING (user_id = current_setting('app.user_id', true)::uuid)
       - The second arg `true` to current_setting means "return NULL if not set" instead of raising error
       - USING clause applies to SELECT, UPDATE, DELETE
    4. CREATE POLICY "user_isolation_insert" ON {table} FOR INSERT WITH CHECK (user_id = current_setting('app.user_id', true)::uuid)
       - Separate INSERT policy needed because USING doesn't apply to INSERT

    NOTE on superuser bypass: The PostgreSQL blitz service account will have BYPASSRLS only if explicitly granted. Do NOT grant BYPASSRLS in this migration — backend queries set app.user_id via SET LOCAL, so they pass the policy. Celery workers also set app.user_id before queries per the same mechanism.

    NOTE on aiosqlite tests: RLS policies are PostgreSQL-only. The migration must check if we're running on SQLite and skip RLS DDL in that case. Use op.get_bind().dialect.name check OR wrap in:
    ```python
    bind = op.get_bind()
    if bind.dialect.name == "postgresql":
        op.execute(sa.text("ALTER TABLE memory_facts ENABLE ROW LEVEL SECURITY"))
        # ... etc
    ```

    downgrade() function: disable RLS and drop policies:
    ```sql
    DROP POLICY IF EXISTS "user_isolation" ON {table};
    DROP POLICY IF EXISTS "user_isolation_insert" ON {table};
    ALTER TABLE {table} DISABLE ROW LEVEL SECURITY;
    ALTER TABLE {table} NO FORCE ROW LEVEL SECURITY;
    ```
    Also wrap downgrade in postgresql dialect check.

    ALSO update backend/core/db.py:
    - Add async function `set_rls_user_id(session: AsyncSession, user_id: UUID) -> None`
    - Body: await session.execute(sa.text("SET LOCAL app.user_id = :uid"), {"uid": str(user_id)})
    - This function is called by route handlers and Celery tasks before any user-scoped DB query
    - Add import: from uuid import UUID
    - Add import: import sqlalchemy as sa (if not already present)
    - Export set_rls_user_id in __all__ or at module level (no __all__ needed, just define at top level)

    NOTE: Do NOT actually run the migration against the database in this task — the migration file is the artifact. Migrations can only be applied via docker exec (no .env on host per known gotcha). The migration will be applied during integration testing by the executor.
