      ```typescript
      interface RegistryDetailLayoutProps {
        entry: RegistryEntry;
        backHref: string;
        backLabel: string;
        tabs: { id: string; label: string }[];

      }
      ```
    - Renders: breadcrumb (back link / entry name), header (displayName or name, description, status badge, timestamps), tab bar, children slot, StickySaveBar (if onSave provided and hasChanges)
    - Adds `beforeunload` event listener when `hasChanges` is true — warns user about leaving
    - Status badge colors: active=green, disabled=gray, deprecated=yellow, pending_review=orange, draft=blue
    - Pattern taken from existing skill detail page header/tabs but extracted as reusable

    **4. Create `frontend/src/lib/registry-schemas.ts`:**
    - Zod schemas for form validation of each registry type's editable fields:
    - `agentFormSchema`: displayName (string, optional), description (string, optional), systemPrompt (string, optional), modelAlias (string, optional — one of "blitz/master", "blitz/fast", "blitz/coder", "blitz/summarizer"), routingKeywords (array of strings, optional), status (enum: active/disabled/deprecated)
    - `toolFormSchema`: displayName (string, optional), description (string, optional), handlerType (enum: backend/mcp/sandbox), handlerModule (string, optional), handlerFunction (string, optional), mcpToolName (string, optional), sandboxRequired (boolean), status (enum)
    - `mcpServerFormSchema`: displayName (string, optional), description (string, optional), url (string, url format, required), authToken (string, optional — only for updates, never pre-filled), status (enum)
    - `skillFormSchema`: displayName (string, optional), description (string, optional), skillType (enum: instructional/procedural), instructionMarkdown (string, optional — required when skillType=instructional), slashCommand (string, optional), category (string, optional), tags (array of strings, optional), status (enum)
    - Each schema exports a type via `z.infer<typeof schema>`
    - Helper function `validateField(schema, fieldName, value)` returning `string | null` (error message or null) for on-blur validation

    **5. Add MCP connection test endpoint to `backend/api/routes/mcp_servers.py`:**
    - Add a new Pydantic model `McpTestRequest` with fields: `url: str`, `auth_token: str | None = None`
    - Add `POST /api/admin/mcp-servers/test` endpoint (before the `/{server_id}` routes to avoid path conflicts)
    - The endpoint:
      1. Creates an SSE connection to `{url}/sse` (matching MCP HTTP+SSE transport)
      2. Sends `tools/list` JSON-RPC request
      3. Returns `{ "success": bool, "latency_ms": int, "tool_count": int | None, "error": str | None, "hint": str | None }`
    - Timeout: 10 seconds
    - On connection refused: hint = "Check that the MCP server is running and the URL is correct"
    - On SSL error: hint = "SSL certificate verification failed — check TLS configuration"
    - On timeout: hint = "Server did not respond within 10 seconds"
    - Requires admin permission (use existing `_require_admin` dependency)

    **6. Add test for MCP test endpoint in `backend/tests/api/test_mcp_server_routes.py`:**

    - Test that the endpoint requires admin authentication (returns 401/403 without token)
