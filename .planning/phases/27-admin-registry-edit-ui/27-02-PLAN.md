    **Data loading:**
    - Use `useParams<{ id: string }>()` to get entry ID
    - Fetch `GET /api/registry/${id}` on mount, map with `mapSnakeToCamel`
    - Show loading spinner, error state with back link to `/admin/agents`

    **Tabs:** Overview, Config, Permissions (per CONTEXT.md decision)

    **Overview tab:**
    - Read-only fields in a `dl` grid (2 cols): ID (mono), Name (read-only text), Status, Owner ID, Created, Updated
    - Editable fields: Display Name (text input), Description (textarea)
    - All editable fields validate on blur using `validateField(agentFormSchema, fieldName, value)` — show red text error below field

    **Config tab — type-specific fields extracted from `entry.config`:**
    - System Prompt: textarea (tall, min 6 rows) — value from `config.system_prompt`
    - Model Alias: dropdown select with options: blitz/master, blitz/fast, blitz/coder, blitz/summarizer — value from `config.model_alias`
    - Routing Keywords: comma-separated text input — value from `config.routing_keywords` (array)
    - Handler Module: text input (read-only, gray background) — from `config.handler_module`
    - Handler Function: text input (read-only, gray background) — from `config.handler_function`
    - "Advanced (raw JSON)" collapsible section at bottom showing remaining config keys as JSON textarea — collapsed by default

    **Permissions tab:**
    - Placeholder for now: "Permission management coming soon" message
    - This tab exists for layout consistency; full permissions UI is out of scope for this phase

    **Change tracking:**
    - Keep a `formData` state initialized from entry on load
    - Compare `formData` to original entry to detect `hasChanges`
    - Pass `hasChanges`, `saving`, `onSave`, `onDiscard` to `RegistryDetailLayout`

    **Save logic:**
    - Run full Zod validation (`agentFormSchema.safeParse`) before save
    - If validation fails, show all inline errors, do not send request
    - Build `RegistryEntryUpdate` payload: `{ display_name: formData.displayName, description: formData.description, config: { ...existingConfig, system_prompt: formData.systemPrompt, model_alias: formData.modelAlias, routing_keywords: formData.routingKeywords }, status: formData.status }`
    - PUT to `/api/registry/${id}` with JSON body (snake_case keys)
    - On success: show success toast (use `alert()` if no toast lib exists), update entry state, reset hasChanges
    - On error: show error toast with message, keep form editable

    **Router guard:**
    - Use Next.js `useRouter` + `beforeunload` (handled by RegistryDetailLayout) for unsaved changes warning
