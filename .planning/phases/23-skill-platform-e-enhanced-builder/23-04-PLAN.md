Critical pitfall (from RESEARCH.md):
- `review_skill` endpoint (admin_skills.py line ~632) raises HTTP 409 if skill.status != "pending_review"
- Builder save endpoint MUST set status='pending_review' AND is_active=False BEFORE the frontend calls review
- For existing active skills re-scanned to review/reject: use UPDATE (not INSERT), set status='pending_review'

- "Approve & Activate" button only visible to users with registry:manage permission
