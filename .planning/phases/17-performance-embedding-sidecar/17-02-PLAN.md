# Phase 17-02: Instrumentation — `timed()` and Critical Path Logging

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add `duration_ms` logging to 7 critical paths via a reusable `timed()` context manager.

**Architecture:** `timed()` is a thin context manager in `core/logging.py` — no middleware, no monkey-patching. It wraps the target code block, starts a monotonic timer at entry, and logs `duration_ms` in the `finally` block so it fires even if the block raises.

**Tech Stack:** Python `contextlib` (stdlib), `structlog` (already in deps)

**Requirements:** PERF-07

**Depends on:** 17-01

from contextlib import contextmanager
from collections.abc import Generator

**Step 2: Add `timed()` after `get_audit_logger()`**

```python
@contextmanager
def timed(logger: structlog.stdlib.BoundLogger, event: str, **ctx: object) -> Generator[None, None, None]:
    """
    Context manager that logs `event` with `duration_ms` after the block exits.

    Logs even if the block raises an exception (duration up to the raise point).

    Usage:
        with timed(logger, "memory_search", user_id=str(user_id)):
            results = await search_facts(...)

    Args:
        logger: structlog logger (from structlog.get_logger(__name__))
        event: Log event name (e.g. "memory_search", "tool_execution")
        **ctx: Extra key-value pairs to include in the log entry
    """
    t0 = time.monotonic()
    try:
        yield
    finally:
        logger.info(event, duration_ms=round((time.monotonic() - t0) * 1000), **ctx)
```
