<objective>
Build the Telegram channel sidecar as an isolated Docker service that translates between Telegram Bot API and InternalMessage format. The sidecar handles webhook registration, inbound message translation (including @mention detection for groups), outbound message formatting (MarkdownV2 + InlineKeyboard), and typing indicators.

Purpose: Enable users to interact with the Blitz agent via Telegram bot with full text messaging and inline button support.

Output: Telegram sidecar FastAPI service, Telegram API wrapper, Dockerfile, Docker Compose service, and tests.

Key decisions from CONTEXT.md (locked):
- Sidecar architecture: standalone FastAPI container on port 9001
- /webhook (inbound) and /send (outbound) endpoints
- Group behavior: respond to @mentions only, not all messages
- Threaded replies in groups (reply to original message)
- Typing indicator: sendChatAction (typing) before agent processing
- MarkdownV2 for text formatting
- Message truncation at 4096 chars with "... (response truncated)"
- Attachments: text-only MVP — respond "I can only process text messages" if user sends media
- Button overflow: up to ~100 buttons (Telegram's limit), drop excess silently
</context>

  <action>
    Create the Telegram sidecar as a standalone Python project:

    1. **pyproject.toml**: project name "blitz-telegram-gateway", Python >=3.12, dependencies: fastapi, uvicorn[standard], httpx, structlog, pydantic>=2.0. Dev deps: pytest, pytest-asyncio, httpx (for TestClient).

    2. **TelegramAPI wrapper** (`telegram_api.py`):
       - Class with `__init__(self, bot_token: str)` — stores token, builds base_url `https://api.telegram.org/bot{token}`
       - `async send_message(chat_id, text, parse_mode="MarkdownV2", reply_to_message_id=None, reply_markup=None)` — POST to /sendMessage
       - `async send_chat_action(chat_id, action="typing")` — POST to /sendChatAction
       - `async set_webhook(url, allowed_updates=None)` — POST to /setWebhook
       - `escape_markdown_v2(text)` — static method to escape special chars for MarkdownV2: `_*[]()~>#+-=|{}.!`
       - `build_inline_keyboard(actions: list[dict])` — builds InlineKeyboardMarkup JSON from MessageAction-like dicts
       - All methods use httpx.AsyncClient internally
       - Log errors with structlog, NEVER log the bot token

    3. **FastAPI app** (`main.py`):
       - `POST /webhook`: receive Telegram Update JSON
         - Extract message or callback_query
         - For messages: extract from.id as external_user_id, chat.id as external_chat_id, text
         - For callback_query: extract from.id, message.chat.id, callback_data → metadata.callback_data
         - Group detection: if chat.type in ("group", "supergroup"), set is_group=True
         - @mention filter: if is_group and bot username not in text entities, skip (return 200 without forwarding)
         - Attachment detection: if message has photo/document/audio/video but no text, respond "I can only process text messages at the moment."
         - Build InternalMessage and POST to BACKEND_URL/api/channels/incoming
         - Send typing indicator (sendChatAction) before forwarding to backend
         - Include display_name in metadata from from.first_name + from.last_name
       - `POST /send`: receive InternalMessage JSON
         - If actions present: build InlineKeyboardMarkup
         - Escape text for MarkdownV2
         - Truncate at 4096 chars with "... (response truncated)" if needed
         - Call telegram_api.send_message with reply_to_message_id if metadata.reply_to_message_id present
       - `GET /health`: return {"status": "ok", "service": "telegram-gateway"}
       - On startup: call set_webhook with TELEGRAM_WEBHOOK_URL env var (skip if not set)
       - Config from env: TELEGRAM_BOT_TOKEN, TELEGRAM_WEBHOOK_URL, BACKEND_URL (default http://backend:8000)

    4. **Tests**:
       - `test_webhook.py`: test_text_message_forwarded (mock httpx POST to backend), test_callback_query_forwarded, test_group_mention_forwarded, test_group_non_mention_ignored, test_attachment_rejected
       - `test_send.py`: test_send_text_message (mock Telegram API), test_send_with_inline_keyboard, test_long_message_truncated
       - Use TestClient from FastAPI, mock httpx calls to Telegram API and backend
