3. Create `backend/skill_export/exporter.py`:
   - `def build_skill_zip(skill: SkillDefinition) -> BytesIO:`
     - Create an in-memory zip using `zipfile.ZipFile(BytesIO(), 'w', zipfile.ZIP_DEFLATED)`
     - Base directory: skill.name (sanitized: lowercase, hyphens for spaces)

     - **SKILL.md generation:**
       - Build YAML frontmatter dict:
         ```python
         frontmatter = {
             "name": skill.name,
             "description": skill.description or "",
         }
         # Add metadata section
         metadata = {
             "author": "blitz-agentos",
             "version": skill.version,
             "skill_type": skill.skill_type,
             "exported_at": datetime.utcnow().isoformat() + "Z",
         }
         if skill.slash_command:
             metadata["slash_command"] = skill.slash_command
         if skill.source_type:
             metadata["source_type"] = skill.source_type
         frontmatter["metadata"] = metadata
         ```
       - Serialize frontmatter as YAML (use `yaml.dump` with default_flow_style=False)
       - Build SKILL.md content: `---\n{yaml}\n---\n\n{instruction_markdown or ""}`
       - Write to `{skill.name}/SKILL.md` in zip

     - **scripts/procedure.json** (if procedural):
       - If `skill.procedure_json` is not None:
         - Write `json.dumps(skill.procedure_json, indent=2)` to `{skill.name}/scripts/procedure.json`

     - **references/schemas.json** (if schemas exist):
       - If `skill.input_schema` or `skill.output_schema`:
         - Build dict: `{"input_schema": skill.input_schema, "output_schema": skill.output_schema}`
         - Write to `{skill.name}/references/schemas.json`

     - Seek BytesIO to 0 and return

1. Fix admin proxy for binary responses in `frontend/src/app/api/admin/[...path]/route.ts`:
   - Currently the proxy does `const responseBody = await backendResponse.text()` and returns `new NextResponse(responseBody, ...)`.
   - This breaks binary responses (zip files). Fix:
     - Check the response Content-Type header
     - If Content-Type is `application/zip` or `application/octet-stream`:
       - Use `const responseBody = await backendResponse.arrayBuffer()` instead of `.text()`
       - Return `new NextResponse(responseBody, { status, headers: { "Content-Type": contentType, "Content-Disposition": backendResponse.headers.get("Content-Disposition") ?? "" } })`
     - Else: keep existing `.text()` behavior for JSON responses
   - Also forward the `Content-Disposition` header from backend response (for filename)
