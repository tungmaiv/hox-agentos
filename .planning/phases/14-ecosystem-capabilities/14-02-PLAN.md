<objective>
Implement the OpenAPI Bridge that lets admins connect any OpenAPI-described service as callable tools. The backend parses specs, the admin wizard lets users select endpoints, and a runtime proxy dispatches tool calls as HTTP requests.

Purpose: Turns any REST API with an OpenAPI spec into MCP-like tools — no Docker containers, no custom code. Immediate value for integrating third-party services.

Output: openapi_bridge module (parser, proxy, service, routes, schemas), tool_registry dispatch for openapi_proxy, frontend wizard component, tests.

**Note:** ECO-02 in ROADMAP says "User provides OpenAPI URL" — per CONTEXT.md decision, this is intentionally implemented as an admin-only wizard (not a user-facing skill or chat interaction). The admin panel flow matches the Phase 12 admin desk pattern. This override is documented in CONTEXT.md: "API-to-MCP interaction is admin panel only".
</objective>

1. Create `backend/openapi_bridge/__init__.py` — empty.

2. Create `backend/openapi_bridge/schemas.py`:
   ```python
   class ParameterInfo(BaseModel):
       name: str
       location: str  # "path" | "query" | "header"
       required: bool
       schema_type: str  # "string" | "integer" | "boolean" | "number" | "array" | "object"
       description: str | None = None

   class EndpointInfo(BaseModel):
       operation_id: str | None
       method: str  # GET, POST, PUT, DELETE, PATCH
       path: str
       summary: str | None
       description: str | None
       tags: list[str]
       parameters: list[ParameterInfo]
       request_body_schema: dict[str, Any] | None = None
       deprecated: bool = False

   class ParseRequest(BaseModel):
       url: str  # URL to fetch OpenAPI spec from

   class ParseResponse(BaseModel):
       base_url: str
       title: str | None
       version: str | None
       endpoints: list[EndpointInfo]
       tag_groups: dict[str, list[int]]  # tag -> endpoint indices for UI grouping

   class RegisterRequest(BaseModel):
       server_name: str
       base_url: str
       spec_url: str
       selected_endpoints: list[EndpointInfo]
       auth_type: str  # "bearer" | "api_key" | "basic" | "none"
       auth_value: str | None = None  # API key, token, or "user:pass" for basic
       auth_header: str | None = None  # custom header name for api_key type

   class RegisterResponse(BaseModel):
       server_id: str
       tools_created: int
   ```

3. Create `backend/openapi_bridge/parser.py`:
   - `async def fetch_and_parse_openapi(url: str) -> ParseResponse:`
   - Fetch spec via `httpx.AsyncClient` (30s timeout, follow redirects)
   - Detect JSON or YAML (try json.loads first, then yaml.safe_load if ImportError use pyyaml)
   - Validate it's an OpenAPI 3.x spec (check `openapi` field starts with "3.")
   - Extract `servers[0].url` as base_url (or derive from spec URL if no servers)
   - Extract `info.title` and `info.version`
   - For each path + method in `paths`:
     - Skip if `deprecated: true`
     - Build `EndpointInfo` with operation_id, method, path, summary, tags, parameters
     - For parameters: extract name, in (→ location), required, schema.type
     - For requestBody: extract `content.application/json.schema` as request_body_schema
   - Build tag_groups dict: for each unique tag, list the endpoint indices
   - Endpoints without tags go into a "default" group
   - Return ParseResponse
   - Add `pyyaml` as dependency if not already present (check pyproject.toml first — likely already there via other deps)

4. Create `backend/openapi_bridge/proxy.py`:
   - `async def call_openapi_tool(tool_config: dict, arguments: dict, api_key: str | None, auth_type: str = "bearer", auth_header: str | None = None) -> dict:`
   - Build URL: `base_url + path` with path parameter substitution from arguments
   - Separate arguments into path_params, query_params, header_params, body based on config parameter locations
   - Build headers: apply auth based on auth_type:
     - "bearer": `Authorization: Bearer {api_key}`
     - "api_key": `{auth_header or "X-API-Key"}: {api_key}`
     - "basic": `Authorization: Basic {base64(api_key)}`  (api_key is "user:pass")
     - "none": no auth header
   - Execute via `httpx.AsyncClient` (30s timeout)
   - On 2xx: return response JSON (or `{"text": response.text}` if not JSON)
   - On non-2xx: return `{"error": True, "status": status_code, "detail": response.text[:500]}`
   - Log via structlog: user_id, tool_name, method, path, status, duration_ms

5. Create `backend/openapi_bridge/service.py`:
   - `async def register_openapi_endpoints(server_name: str, base_url: str, spec_url: str, endpoints: list[EndpointInfo], auth_type: str, auth_value: str | None, auth_header: str | None, session: AsyncSession) -> RegisterResponse:`
   - Create `McpServer` row: name=server_name, url=base_url, openapi_spec_url=spec_url
   - If auth_value provided: encrypt with `encrypt_token()`, store in `auth_token` field
   - Store auth config in a new `config_json` on the McpServer (auth_type + auth_header)
     - Actually, McpServer doesn't have config_json. Store auth metadata in ToolDefinition.config_json instead.
   - For each endpoint:
     - Generate tool name: `{server_name}.{operation_id or method_path}` (sanitize: lowercase, replace non-alphanumeric with underscore)
     - Build `input_schema` from endpoint parameters + requestBody
     - Build `config_json`: `{"method": endpoint.method, "path": endpoint.path, "base_url": base_url, "parameters": [...], "auth_type": auth_type, "auth_header": auth_header}`
     - Create `ToolDefinition` row: name, display_name=summary, description, handler_type="openapi_proxy", input_schema, config_json, mcp_server_id=server.id
   - Call `invalidate_tool_cache()` to refresh registry
   - Return RegisterResponse with server_id and tools_created count

6. Create `backend/openapi_bridge/routes.py`:
   - `router = APIRouter(prefix="/api/admin/openapi", tags=["admin-openapi"])`
   - Both endpoints require JWT + `registry:manage` permission (use existing deps: `get_current_user` + RBAC check)
   - `POST /parse`: accept ParseRequest, call fetch_and_parse_openapi, return ParseResponse
   - `POST /register`: accept RegisterRequest, call register_openapi_endpoints, return RegisterResponse

7. Register router in `backend/main.py` (routers are registered via `app.include_router()` in `main.py` — NOT in `routes/__init__.py`).

8. Update `backend/gateway/tool_registry.py` — add `openapi_proxy` dispatch:
   - In the tool dispatch logic (wherever handler_type is checked), add a case for `handler_type == "openapi_proxy"`:
     - Load `config_json` from the tool definition
     - Load API key: find the associated `McpServer` via `mcp_server_id`, decrypt `auth_token`
     - Call `call_openapi_tool(config_json, arguments, api_key, auth_type, auth_header)`
   - If tool_registry.py doesn't have dispatch logic (it's just a registry), check where tool dispatch actually happens (likely in `gateway/runtime.py` or agent tool execution) and add the openapi_proxy case there instead.
