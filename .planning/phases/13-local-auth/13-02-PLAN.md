**1. Add Credentials provider to `frontend/src/auth.ts`:**

Import `Credentials` from `next-auth/providers/credentials`. Add it as a second provider alongside the existing Keycloak provider:

```ts
Credentials({
  credentials: {
    username: { label: "Username", type: "text" },
    password: { label: "Password", type: "password" },
  },
  async authorize(credentials) {
    if (!credentials?.username || !credentials?.password) return null;
    const backendUrl = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000";
    const res = await fetch(`${backendUrl}/api/auth/local/token`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        username: credentials.username,
        password: credentials.password,
      }),
    });
    if (!res.ok) return null;
    const { access_token } = await res.json() as { access_token: string };
    // Decode JWT payload (no verification — backend already verified)
    const payload = JSON.parse(
      Buffer.from(access_token.split(".")[1], "base64url").toString()
    );
    return {
      id: payload.sub,
      name: payload.preferred_username,
      email: payload.email,
      accessToken: access_token,
      realmRoles: payload.realm_roles ?? [],
    };
  },
})
```

**Update the `jwt()` callback** in auth.ts:
- When `account?.provider === "credentials"` (initial sign-in from Credentials provider), the `user` object has `accessToken` and `realmRoles`. Store them:
  ```ts
  if (account?.provider === "credentials" && user) {
    const u = user as { accessToken?: string; realmRoles?: string[] };
    return {
      ...token,
      accessToken: u.accessToken,
      realmRoles: u.realmRoles ?? [],
      // Local tokens: 8-hour expiry from iat, no refresh token
      expiresAt: Math.floor(Date.now() / 1000) + 8 * 3600,
      authProvider: "credentials",
    };
  }
  ```
- In the existing Keycloak sign-in block (`if (account)`), add `authProvider: "keycloak"`.
- In the token refresh block: only attempt refresh when `authProvider === "keycloak"` (local tokens have no refresh_token — user re-authenticates after 8-hour expiry).
- When local token expires (`authProvider === "credentials"` and past expiry), set `error: "SessionExpired"` so the client can redirect to /login.

**Update the `session()` callback:**
- Propagate `realmRoles` to session: `(session as any).realmRoles = token.realmRoles ?? [];`
- This is needed for the admin layout RBAC check.

This is NOT strictly required since the Credentials provider calls the backend directly from the server side. However, create it as a pass-through proxy for any future client-side direct login calls:

import { NextRequest, NextResponse } from "next/server";

const BACKEND_URL = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000";

export async function POST(req: NextRequest): Promise<NextResponse> {
  const body = await req.json();
  const res = await fetch(`${BACKEND_URL}/api/auth/local/token`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(body),
  });
  const data = await res.json();
  return NextResponse.json(data, { status: res.status });
}

**2. Create Next.js API proxy routes for all backend endpoints:**

Follow the existing proxy pattern from `frontend/src/app/api/admin/credentials/route.ts` — use `getAccessToken()` helper that reads from the next-auth session.

Create a shared helper function `getAccessToken()` at the top of each proxy (or in a shared utility if one already exists). Pattern:
```ts
import { auth } from "@/auth";
import { NextRequest, NextResponse } from "next/server";

const API = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000";

async function getAccessToken(): Promise<string | null> {
  const session = await auth();
  if (!session) return null;
  return (session as unknown as Record<string, unknown>).accessToken as string | null;
}
```
