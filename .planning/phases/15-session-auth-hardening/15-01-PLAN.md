Per user decision: **allowlist approach** — everything is protected by default. Public routes explicitly listed.

The middleware must:

1. **Define public routes allowlist:**
   ```typescript
   const PUBLIC_PATHS = [
     "/login",
     "/api/auth",    // next-auth routes (/api/auth/callback/*, /api/auth/session, etc.)
   ];
   const PUBLIC_PREFIXES = [
     "/_next",       // Next.js static assets
     "/favicon.ico",
   ];
   ```

2. **Use Next.js `config.matcher`** to exclude static files efficiently:
   ```typescript
   export const config = {
     matcher: [
       // Match all paths except static files and _next internals
       "/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp|ico)$).*)",
     ],
   };
   ```

3. **Check if path is public** — if yes, let through with `NextResponse.next()`.

4. **For protected paths — verify session exists:**
   The next-auth v5 session cookie name is `authjs.session-token` (or `__Secure-authjs.session-token` in production with HTTPS). Read this cookie from the request.

   Use `getToken` from `next-auth/jwt` which is Edge Runtime compatible in next-auth v5. This is simpler and more reliable than raw `jose.jwtVerify()` because next-auth encrypts the session cookie with its own secret (not the Keycloak/local JWT directly).

   ```typescript
   import { getToken } from "next-auth/jwt";

   const token = await getToken({ req: request });
   if (!token) {
     // No valid session → redirect to /login with callbackUrl
     const loginUrl = new URL("/login", request.url);
     loginUrl.searchParams.set("callbackUrl", request.nextUrl.pathname);
     return NextResponse.redirect(loginUrl);
   }
   ```

   This is the correct approach because next-auth v5 encrypts the JWT in the cookie using `NEXTAUTH_SECRET` — raw `jose.jwtVerify()` cannot decrypt it. `getToken()` from `next-auth/jwt` handles decryption and is Edge Runtime compatible.

   **Important:** `jose` is still needed as a dependency because `next-auth/jwt` internally uses `jose` for Edge Runtime JWT operations. If `jose` is not installed, `getToken()` will fail at runtime.

5. **Check for session errors** — if the token has `error` field set (e.g., `"SessionExpired"` or `"RefreshAccessTokenError"`), redirect to login:
   ```typescript
   if (token.error) {
     const loginUrl = new URL("/login", request.url);
     loginUrl.searchParams.set("callbackUrl", request.nextUrl.pathname);
     loginUrl.searchParams.set("error", token.error as string);
     return NextResponse.redirect(loginUrl);
   }
   ```

6. **Set security headers on response** (AUTH-03 cookie hardening):
   The actual cookie attributes (HttpOnly, Secure, SameSite) are controlled by next-auth's cookie config in `auth.ts`, not in middleware. However, add a comment documenting that the cookie config must be verified in auth.ts.

7. **Add callbackUrl support to the redirect** — per user decision, return user to their previous page after re-login via `?callbackUrl=<current-path>` on /login redirect.

**Full middleware structure:**
```typescript
import { NextRequest, NextResponse } from "next/server";
import { getToken } from "next-auth/jwt";

// Public paths that don't require authentication
const PUBLIC_PATHS = ["/login"];
const PUBLIC_PATH_PREFIXES = ["/api/auth/"];

function isPublicPath(pathname: string): boolean {
  if (PUBLIC_PATHS.includes(pathname)) return true;
  return PUBLIC_PATH_PREFIXES.some((prefix) => pathname.startsWith(prefix));
}

export async function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // Allow public paths
  if (isPublicPath(pathname)) {
    return NextResponse.next();
  }

  // Verify session token
  const token = await getToken({ req: request });

  if (!token) {
    const loginUrl = new URL("/login", request.url);
    loginUrl.searchParams.set("callbackUrl", pathname);
    return NextResponse.redirect(loginUrl);
  }

  // Check for session errors (expired, refresh failed)
  if (token.error) {
    const loginUrl = new URL("/login", request.url);
    loginUrl.searchParams.set("callbackUrl", pathname);
    loginUrl.searchParams.set("error", token.error as string);
    return NextResponse.redirect(loginUrl);
  }

  return NextResponse.next();
}

export const config = {
  matcher: [
    "/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp|ico)$).*)",
  ],
};
```
