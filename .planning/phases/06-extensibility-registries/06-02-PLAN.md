    Modify `backend/security/rbac.py`:

    1. **Keep ROLE_PERMISSIONS dict** as fallback (used when DB is empty or during testing without DB).

    2. **Add module-level cache:**
    ```python
    _permission_cache: dict[str, set[str]] = {}
    _cache_timestamp: float = 0.0
    _CACHE_TTL: float = 60.0
    ```

    3. **Convert has_permission() to async with session parameter:**
    ```python
    async def has_permission(user: UserContext, permission: str, session: AsyncSession | None = None) -> bool:
    ```
    - If session is None (backward compat), fall back to ROLE_PERMISSIONS dict (existing behavior)
    - If session provided, check `_permission_cache` (refresh from DB if expired)
    - Cache refresh: `SELECT role, permission FROM role_permissions` -> build dict[str, set[str]]
    - If DB returns 0 rows, fall back to ROLE_PERMISSIONS dict (safety net)

    4. **Add check_artifact_permission():**
    ```python
    async def check_artifact_permission(
        user: UserContext, artifact_type: str, artifact_id: UUID, session: AsyncSession
    ) -> bool:
    ```
    - Query: `SELECT allowed FROM artifact_permissions WHERE artifact_type=:type AND artifact_id=:id AND role IN :roles AND status='active'`
    - Only check permissions with status='active' (not 'pending' -- staged model)
    - No rows = default ALLOW (same pattern as existing tool_acl)
    - Any row with `allowed=False` for any of user's roles = DENY
    - Any row with `allowed=True` and no denials = ALLOW
    - Also check user_artifact_permissions for per-user overrides: if user-level override exists, it takes precedence over role-level

    5. **Add invalidate_permission_cache():** Resets `_cache_timestamp` to 0 so next call refreshes from DB.

    6. **Keep sync get_permissions()** for backward compat if any callers use it synchronously.

    - Test: employee has registry:manage = False
    - Test: it-admin has registry:manage = True
    - Test: unknown_role has chat = False
    - Test: artifact permission default allow (no rows)
    - Test: artifact permission explicit deny (allowed=False row)
    - Test: artifact permission staged pending row is ignored (only status='active' checked)
    - Test: user_artifact_permission override takes precedence over role-level
    - Test: cache invalidation triggers refresh
