In `frontend/src/app/admin/skills/page.tsx`, fix the `displayItems` filter at lines 50-57.

Current (wrong):
```typescript
const displayItems = showPendingOnly
  ? items.filter(
      (s) =>
        s.status === "active" &&
        s.securityScore !== null &&
        s.securityScore < 70
    )
  : items;
```

Replace with (correct):
```typescript
const displayItems = showPendingOnly
  ? items.filter((s) => s.status === "pending_review")
  : items;
```

Rationale: "Pending Review" is a distinct status value set on skills that have been
imported and await admin review. The previous logic was filtering on active+low-score,
which is conceptually different (those are approved but risky skills, not pending review).
The button label says "Pending Review" — it should filter by that exact status.

No other changes to this file. The extraColumns, card rendering, and create form are
all correct and should not be touched.
