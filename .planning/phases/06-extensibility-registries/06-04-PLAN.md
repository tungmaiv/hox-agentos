  <action>
    **Migrate tool_registry.py from in-memory dict to DB-backed:**

    1. Replace `_registry: dict[str, dict] = {}` with a DB-backed cache:
    ```python
    _tool_cache: dict[str, dict] = {}
    _tool_cache_timestamp: float = 0.0
    _TOOL_CACHE_TTL: float = 60.0
    ```

    2. Add `async def _refresh_tool_cache(session: AsyncSession)`:
    - Query `SELECT * FROM tool_definitions WHERE status='active'`
    - Convert each ToolDefinition row to dict (name, handler_type, handler_module, handler_function, etc.)
    - Store in `_tool_cache`

    3. Convert `get_tool(name)` to `async def get_tool(name: str, session: AsyncSession) -> dict | None`:
    - Check TTL, refresh if expired
    - Return from cache

    4. Convert `register_tool()` to `async def register_tool(session: AsyncSession, **kwargs)`:
    - Upsert into tool_definitions table (SELECT + INSERT or UPDATE)
    - Invalidate cache

    5. Add `list_tools(session) -> list[dict]` -- returns all active tools from cache.

    6. Add `invalidate_tool_cache()` -- resets timestamp to 0.

    7. **Keep backward compat:** If `session=None` is passed to `get_tool`, fall back to `_tool_cache` without refresh (stale-or-empty). This prevents breaking callers that haven't been updated yet.

    **Update all callers of get_tool/register_tool** to pass session. Search `gateway/tool_registry` imports across the codebase.

    **Update mcp/registry.py:**
    - `MCPToolRegistry.refresh()`: Upsert discovered MCP tools into `tool_definitions` table (handler_type="mcp", mcp_server_id, mcp_tool_name) instead of calling old `register_tool()` dict API
    - Filter MCP servers by `status == 'active'` instead of `is_active == True`
    - When a server is disabled, evict its client from `_clients` dict

    **MCP server routes (mcp_servers.py):**
    - Add `GET /{server_id}/health` endpoint: look up server URL, make httpx GET to `{url}/health` with 5s timeout, return `{"reachable": true/false, "latency_ms": N}`
    - Add `PATCH /{server_id}/status` endpoint: update status column, invalidate tool cache, evict client from MCPToolRegistry._clients if disabled

    **Tests:**
    - test_tool_registry_db.py: SQLite fixture with seeded tool_definitions rows. Test get_tool returns active tools, returns None for disabled, register_tool upserts, list_tools only active, cache works, invalidation works.
    - test_mcp_evolution.py: Test health-check endpoint (mock httpx), test status patch updates DB, test refresh skips disabled servers.
