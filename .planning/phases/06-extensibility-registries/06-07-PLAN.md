    - Simple toggle button between "table" and "cards" view modes
    - Stores preference in localStorage under `admin-view-mode`

    **Artifact table component (frontend/src/components/admin/artifact-table.tsx):**
    - Generic `ArtifactTable<T>` component
    - Columns: name, version, status (badge: green=active, gray=disabled, yellow=deprecated), is_active (checkmark), last_seen_at (relative time), actions
    - Row actions: Edit (opens modal/drawer), Enable/Disable (PATCH status), Activate Version
    - Status filter dropdown, version filter
    - Sortable columns (name, version, status, last_seen_at)

    **Artifact card grid component (frontend/src/components/admin/artifact-card-grid.tsx):**
    - Generic `ArtifactCardGrid<T>` component
    - Cards show: name, display_name, description (truncated), version badge, status badge, last_seen_at
    - Card actions: same as table row actions
    - Responsive grid: 3 cols on lg, 2 on md, 1 on sm

    **MCP status dot (frontend/src/components/admin/mcp-status-dot.tsx):**
    - `McpStatusDot` component: takes `lastSeenAt: string | null`
    - Green dot: last_seen_at < 5 minutes ago
    - Yellow dot: last_seen_at < 30 minutes ago
    - Red dot: last_seen_at > 30 minutes or null (never seen)
    - Tooltip showing exact last_seen_at time

    **Permission matrix (frontend/src/components/admin/permission-matrix.tsx):**
    - Full matrix view: rows = artifacts, columns = roles
    - Each cell: checkbox (allowed/denied)
    - Pending changes shown with yellow highlight
    - "Apply Pending" button calls applyPending()
    - Per-user overrides section below matrix: list of user-specific exceptions with add/remove
    - Uses `useAdminPermissions` hook
