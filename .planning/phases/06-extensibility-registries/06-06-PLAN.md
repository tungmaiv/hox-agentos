    **User skill routes (backend/api/routes/user_skills.py):**

    Router prefix: `/api/skills`, tags: `["skills"]`

    `GET /` -- list skills available to current user:
    1. Query `skill_definitions WHERE status='active'`
    2. For each skill, check `artifact_permissions` for user's roles -- exclude skills where any role is explicitly denied
    3. Return list of SkillListItem (name, display_name, description, slash_command)
    4. Requires `chat` permission (basic user access)

    `POST /{skill_name}/run` -- execute a skill:
    1. Lookup skill by name WHERE status='active'. If not found -> 404.
    2. Check artifact permission for user's roles. If denied -> 403.
    3. Dispatch based on skill_type:
       - `procedural`: Create SkillExecutor, call `executor.run(skill, user_context, session, user_input)`. Return SkillRunResponse.
       - `instructional`: Return the instruction_markdown content (agent will use it as context injection). Return SkillRunResponse(success=True, output=skill.instruction_markdown).
    4. Log execution via structlog audit logger.

    `GET /` (tools listing, if needed): Also add `GET /api/tools` for listing available tools to current user, filtered by role permissions. Same pattern as skills listing. (Referenced in design doc Section 5.2 but may already exist -- check before adding.)

    **Slash command dispatch in master_agent.py:**

    Modify `_pre_route()` to detect `/command` prefix BEFORE existing keyword routing:

    ```python
    # At the start of _pre_route:
    last_msg = state["messages"][-1].content if state["messages"] else ""
    if isinstance(last_msg, str) and last_msg.strip().startswith("/"):
        command = last_msg.strip().split()[0]  # e.g., "/morning_digest"
        # Check if it's a registered skill slash command
        async with async_session() as session:
            result = await session.execute(
                select(SkillDefinition).where(
                    SkillDefinition.slash_command == command,
                    SkillDefinition.status == "active",
                )
            )
            skill = result.scalar_one_or_none()
            if skill:
                return "skill_executor"
    # ... existing keyword routing continues
    ```

    - For instructional: appends instruction_markdown as system message, then continues to master_agent node for LLM processing
    - Edge: skill_executor -> delivery_router

    **Frontend skill hook (frontend/src/hooks/use-skills.ts):**

    ```typescript
    "use client";
    import { useState, useEffect } from "react";

    interface SkillItem {
      name: string;
      displayName: string;
      description: string;
      slashCommand: string | null;
    }

    export function useSkills() {
      const [skills, setSkills] = useState<SkillItem[]>([]);
      const [loading, setLoading] = useState(true);

      useEffect(() => {
        fetch("/api/skills")
          .then((res) => { if (!res.ok) throw new Error(); return res.json(); })
          .then((data: unknown) => {
            // Validate with type guard before setting
            if (Array.isArray(data)) setSkills(data as SkillItem[]);
          })
          .catch(() => setSkills([]))
          .finally(() => setLoading(false));
      }, []);

      return { skills, loading };
    }
    ```
