<objective>
Inject an `allowed_tools` pre-gate into `SkillExecutor._run_tool_step()` that blocks tool calls not listed in the skill's declared `allowed-tools` field. The check fires before Gate 3 ACL (no DB lookup on denied calls). Denials are audit-logged via `get_audit_logger()`. Backwards-compatible: `allowed_tools=None` or `allowed_tools=[]` is permissive.

Purpose: Enforces skill-declared tool permissions at execution time (SKSEC-02), giving skills a fine-grained declaration of which tools they are allowed to use, independent of the user's broader ACL.
Output: Modified `executor.py` + `TestAllowedTools` class in `test_skill_executor.py`.
</objective>
