For FTS, use the existing `refetch` or re-call pattern. Since `useAdminArtifacts` may not support params, apply client-side filter on `items` for category/author/sort, and use a separate effect + local state for FTS results:

Actually, simpler: filter `items` client-side for everything. For FTS, do case-insensitive substring match on name+description (good enough for admin view — admin catalog is small). This avoids modifying `useAdminArtifacts`:

```typescript
const filteredItems = items.filter((item) => {
  const s = debouncedSearch.toLowerCase();
  const matchesSearch = !s ||
    item.name.toLowerCase().includes(s) ||
    (item.description ?? "").toLowerCase().includes(s);
  const matchesCategory = !filterCategory ||
    (item as SkillDefinition).category === filterCategory;
  const matchesAuthor = !filterAuthor ||
    (item as SkillDefinition).createdBy === filterAuthor;
  return matchesSearch && matchesCategory && matchesAuthor;
});

const sortedItems = [...filteredItems].sort((a, b) => {
  if (sortMode === "most_used") {
    return ((b as SkillDefinition).usageCount ?? 0) - ((a as SkillDefinition).usageCount ?? 0);
  }
  if (sortMode === "oldest") {
    return new Date(a.createdAt ?? 0).getTime() - new Date(b.createdAt ?? 0).getTime();
  }
  // newest (default)
  return new Date(b.createdAt ?? 0).getTime() - new Date(a.createdAt ?? 0).getTime();
});
```
