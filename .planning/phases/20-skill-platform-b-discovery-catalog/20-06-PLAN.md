<objective>
Fix admin /skills table sort so the parent-managed sort order (Newest / Oldest / Most Used) actually reorders the table rows.

Purpose: `ArtifactTable` initialises its own `sortField` state to `"name"` and always sorts by that field, overriding the `displayItems` order produced by the parent's sort dropdown. The admin "Most Used" sort option is also a no-op because `SkillDefinition` lacks a `usageCount` field. Three targeted changes fix this:
1. Add `disableInternalSort` prop to `ArtifactTable` — when `true`, skip internal sort, render `items` as-is.
2. Add `usageCount: number` to `SkillDefinition` in `admin-types.ts`.
3. Update the `most_used` branch in `admin/skills/page.tsx` to sort by `usageCount DESC`.
Output: Sort dropdown on admin /skills page reorders table rows correctly.
</objective>
