## Non-Critical Defects (Tech Debt)

### Phase 2: Agent Core and Conversational Chat

**1. BACKEND_URL / NEXT_PUBLIC_API_URL inconsistency**
- 5 frontend server routes use `process.env.BACKEND_URL` (undefined in `.env.local`); falls back to hardcoded `http://localhost:8000`
- 3 files use `NEXT_PUBLIC_API_URL` correctly
- Affected routes: `/api/copilotkit`, `/api/conversations`, and sub-routes
- **Impact:** Works in local dev (hardcoded default matches). Deployment to any custom URL would silently break CopilotKit proxy + conversation sidebar while SSR fetches in chat/page.tsx still work.
- **Fix:** Consolidate all backend calls to use `NEXT_PUBLIC_API_URL` (or a server-only `BACKEND_URL` set in deployment env)

**2. user_instructions.updated_at not auto-updating**
- `core/models/user_instructions.py`: `updated_at` has `server_default=func.now()` but no `onupdate=func.now()`
- Compare: `conversation_title.py` correctly uses `onupdate=func.now()`
- **Impact:** `PUT /api/user/instructions/` returns creation timestamp as `updated_at` after first save (misleading response data). Functional instruction injection is unaffected.
- **Fix:** Add `onupdate=func.now()` to `UserInstructions.updated_at` + add migration

**3. REQUIREMENTS.md traceability table not updated**
- All 51 requirements still show `[ ] Pending` despite phases 1+2 completing their requirements
- AUTH-01–06, AGNT-01, AGNT-02, AGNT-07, MEMO-01, MEMO-05, INTG-04 should be `[x]`
- **Fix:** Update checkboxes manually or add to post-phase automation

**4. ROADMAP.md 01-04-PLAN.md checkbox shows [ ]**
- Line 42 still shows `[ ] 01-04-PLAN.md` despite Phase 1 being complete

**5. OAuth token ingestion flow deferred**
- AES-256-GCM vault fully implemented; GET/DELETE API endpoints protected
- No `POST /api/credentials` endpoint; no OAuth redirect callback
- Phase 3 (Email/Calendar sub-agents) will add the OAuth flow entry point
