### Critical Integration Gap: Workflow Channel Delivery

**Root cause:** `_handle_channel_output_node` in `backend/agents/node_handlers.py` (lines 218-223) builds an `InternalMessage` using `external_user_id` from the workflow's `user_context`. However, `user_context` (built in `workflow_execution.py:99-105`) only contains `user_id`, `email`, `username`, `roles`, `groups` — there is no `external_user_id` or `external_chat_id`.

**Fix required:** When a workflow reaches a `channel_output_node`, it needs to:
1. Query `channel_accounts` table for the owner's linked account on the target channel
2. Use the stored `external_user_id` from that account as the `external_chat_id` for the outbound `InternalMessage`
3. Handle the case where the owner has no linked account on the target channel (skip delivery with warning)

**Affected requirements:** WKFL-03, WKFL-04 (templates with channel output), indirectly WKFL-06/07 (cron/webhook workflows that include channel output nodes)
