### Integration Issues Found

**Issue 1 (Tech Debt) — HITL canvas node visual state broken**
- **File:** `backend/scheduler/tasks/workflow_execution.py` lines 217-221
- **Finding:** `hitl_paused` event published without `node_id` field. Frontend hook (`use-workflow-run.ts:86`) reads `event.node_id` → `undefined`. `nodeStatuses.set(undefined, "awaiting_approval")` is a no-op. The HITL canvas node never shows the amber `awaiting_approval` ring.
- **What works:** Graph pauses, run status = `paused_hitl`, HITL banner with message appears, Approve/Reject buttons functional, resume path works correctly.
- **Fix:** In `workflow_execution.py` after `state_snapshot = await compiled.aget_state(config)`, add `hitl_node_id = state_snapshot.next[0] if state_snapshot.next else None` and include it in the `publish_event` call.
- **Severity:** Non-critical (UX visual state only)

**Issue 2 (Tech Debt) — agent_node and channel_output_node stubs**
- **Files:** `backend/agents/node_handlers.py` — `_handle_agent_node()` and `_handle_channel_output_node()`
- **Finding:** Both return synthetic output. Real sub-agent dispatch and channel delivery deferred post-Phase 4 per plan.
- **Impact:** Morning Digest template summarize node and Telegram delivery node produce stub output. Graph executes correctly — output content is synthetic.
- **Severity:** Non-critical (known, documented in VERIFICATION.md)

**Issue 3 (Tech Debt) — Celery worker roles hardcoded**
- **File:** `backend/scheduler/tasks/workflow_execution.py` lines 102-104
- **Finding:** Worker constructs UserContext with `roles: ["employee"]` because no JWT is present in Celery context. Tools requiring permissions beyond `employee` will 403 at Gate 2 RBAC inside `call_mcp_tool()`.
- **Severity:** Non-critical (known limitation; all current templates use employee-accessible tools)
