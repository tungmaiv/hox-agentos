  <name>Task 1: Install trufflehog via Go and run git history scan</name>
  <files></files>
  <action>
    Install trufflehog using Go's module system and run a git history credential scan.

    Step 1 — Install trufflehog:
    ```bash
    go install github.com/trufflesecurity/trufflehog/v3@latest
    ```
    This installs the binary to `$(go env GOPATH)/bin/trufflehog`. Verify with:
    ```bash
    $(go env GOPATH)/bin/trufflehog --version
    ```

    Step 2 — Run git history scan:
    ```bash
    cd /home/tungmv/Projects/hox-agentos
    $(go env GOPATH)/bin/trufflehog git file://. --only-verified 2>&1 | tee /tmp/trufflehog-scan.txt
    ```

    Use `--only-verified` to suppress false positives (trufflehog attempts to verify credentials against APIs before reporting). This means reported findings are real, live credentials — not just pattern matches.

    Step 3 — Interpret results:
    - If output is empty or "No verified credentials found" → CLEAN. Document this result.
    - If verified credentials are found → Document each finding with: file path, commit hash, secret type. These would be CRITICAL findings requiring immediate rotation. Report them in SUMMARY.md — do NOT attempt to rewrite git history (that is outside plan scope and risks data loss).

    Step 4 — Also run filesystem scan (current working tree, not just history):
    ```bash
    $(go env GOPATH)/bin/trufflehog filesystem /home/tungmv/Projects/hox-agentos --only-verified --exclude-paths=.venv --exclude-paths=node_modules 2>&1 | tee /tmp/trufflehog-filesystem.txt
    ```

    Note: `.dev-secrets` is in `.gitignore` and should not appear in git history. The `.env` files are also gitignored. False positives for test fixture values (e.g., the 32-byte test encryption key in test_isolation.py) are expected with `--no-verification` but should be filtered out by `--only-verified`.
  </action>
