Create `infra/grafana/provisioning/alerting/alert_rules.yml` — alert fires when daily LiteLLM spend exceeds threshold. Threshold defaults to $10/day (reasonable default per CONTEXT.md Claude's Discretion for thresholds). Per RESEARCH.md pitfall 6: folder must be `__Blitz` and folderUid must be `blitz-folder` (matches dashboards.yml provisioning):
```yaml
apiVersion: 1

groups:
  - orgId: 1
    name: blitz-cost-alerts
    folder: __Blitz
    folderUid: blitz-folder
    interval: 5m
    rules:
      - uid: daily_spend_alert
        title: Daily LiteLLM Spend Exceeded
        condition: A
        for: 5m
        data:
          - refId: A
            datasourceUid: blitz-prometheus
            model:
              expr: sum(increase(litellm_spend_metric[24h])) > 10
              type: range
        annotations:
          summary: "Daily LiteLLM spend has exceeded $10"
          description: "Current 24h spend: {{ $value | humanize }}"
        labels:
          severity: warning
        noDataState: NoData
        execErrState: Error
        isPaused: false
```
