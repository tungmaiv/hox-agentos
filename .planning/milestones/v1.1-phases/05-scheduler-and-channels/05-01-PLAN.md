    Create the central gateway class that all channel sidecars will use:

    1. **ChannelGateway** (`backend/channels/gateway.py`):
       - Constructor takes `sidecar_urls: dict[str, str]` (channel -> sidecar URL)
       - `is_pairing_command(msg)`: checks if text starts with "/pair "
       - `resolve_account(msg, db)`: SELECT ChannelAccount WHERE channel=msg.channel AND external_user_id=msg.external_user_id
       - `resolve_or_create_session(account, msg, db)`: find existing or create new ChannelSession; uses external_chat_id or external_user_id as fallback for DMs
       - `generate_pairing_code(user_id, channel, db)`: generates 6-char alphanumeric code (secrets.token_hex(3).upper()[:6]), 10-min expiry; upserts pending channel_account
       - `handle_pairing(msg, db)`: process /pair command — find account by code+channel, check expiry, link external_user_id, set is_paired=True, clear code
       - `handle_inbound(msg, db)`: main entry — check pairing → resolve identity → resolve session → invoke agent (stub for now) → send outbound
       - `send_outbound(msg)`: POST to sidecar /send endpoint via httpx; skip if channel=="web"; retry 3x with exponential backoff (1s, 2s, 4s) per locked decision; log failure after 3 retries
       - `_invoke_agent(msg)`: stub returning echo response — wired to real agent in 05-05
       - `_make_reply(original, text)`: create outbound InternalMessage from inbound
       - Use structlog for all logging; NEVER log credentials

    2. **Tests** (`backend/tests/channels/test_gateway.py`):
       - test_resolve_paired_account: paired account found by (channel, external_user_id)
       - test_resolve_unknown_account_returns_none: unknown returns None

       - test_generate_pairing_code: generates 6-char code, stores in DB with expiry
       - test_handle_pairing_success: /pair command links account, clears code
       - test_handle_pairing_expired_code: expired code rejected
       - test_is_pairing_command: detects /pair prefix, rejects normal messages and None text
