  <action>
    Create the MS Teams sidecar as a standalone Python project:

    1. **pyproject.toml**: project name "blitz-teams-gateway", Python >=3.12, dependencies: fastapi, uvicorn[standard], httpx, structlog, pydantic>=2.0, PyJWT, cryptography. Dev deps: pytest, pytest-asyncio, httpx.
       NOTE: Do NOT use botbuilder-core/botbuilder-integration-aiohttp — these are heavy and designed for Azure Functions. Use direct HTTP calls to Bot Framework Connector API with httpx instead. This keeps the sidecar lightweight and consistent with Telegram/WhatsApp patterns.

    2. **TeamsAPI wrapper** (`teams_api.py`):
       - Class with `__init__(self, app_id: str, app_password: str)`
       - `async get_token()` — POST to `https://login.microsoftonline.com/botframework.com/oauth2/v2.0/token` for Bearer token (client_credentials grant, scope=https://api.botframework.com/.default). Cache token until expiry.
       - `async send_activity(service_url: str, conversation_id: str, activity: dict)` — POST to `{service_url}/v3/conversations/{conversation_id}/activities` with Bearer token
       - `async send_typing(service_url: str, conversation_id: str)` — send typing Activity
       - `async reply_to_activity(service_url: str, conversation_id: str, activity_id: str, activity: dict)` — POST to `{service_url}/v3/conversations/{conversation_id}/activities/{activity_id}` (threaded reply)
       - `build_adaptive_card(text: str, actions: list[dict])` — build Adaptive Card JSON with TextBlock (markdown body) + Action.Submit buttons
       - `async validate_token(auth_header: str)` — validate Bot Framework JWT: fetch JWKS from `https://login.botframework.com/v1/.well-known/openidconfiguration`, verify issuer is `https://api.botframework.com`, verify audience is app_id. For MVP: validate structure but log warning on signature failures (full validation requires JWKS rotation handling).
       - Log errors with structlog, NEVER log app_password or tokens

    3. **FastAPI app** (`main.py`):
       - `POST /webhook`: receive Bot Framework Activity JSON
         - Validate Authorization header via teams_api.validate_token()
         - Extract activity.type — only process "message" and "invoke" types
         - For "message" activities:
           - Extract from.id as external_user_id, conversation.id as external_chat_id, text
           - Channel detection: if channelData.team exists, set is_group=True
           - @mention filter: if is_group, check if entities contain @mention of bot — if not mentioned, skip
           - Strip @mention text from message body (Teams prepends "@BotName " to message)
           - Attachment detection: if attachments present but no text, respond "I can only process text messages"
           - Include display_name from from.name in metadata
           - Include service_url and activity.id in metadata (needed for threaded replies)
           - Build InternalMessage and POST to BACKEND_URL/api/channels/incoming
           - Send typing indicator before forwarding
         - For "invoke" activities (Adaptive Card Action.Submit):
           - Extract value as metadata.callback_data
           - Forward as InternalMessage
       - `POST /send`: receive InternalMessage JSON
         - Extract service_url from metadata (stored during inbound)
         - If actions present: build Adaptive Card with Action.Submit buttons
         - Otherwise: plain text Activity with textFormat="markdown"
         - Truncate at 4096 chars with "... (response truncated)"
         - If metadata.reply_to_activity_id present: use reply_to_activity (threaded)
         - Otherwise: use send_activity
       - `GET /health`: return {"status": "ok", "service": "teams-gateway"}
       - Config from env: TEAMS_APP_ID, TEAMS_APP_PASSWORD, BACKEND_URL (default http://backend:8000)

    4. **Tests**:
       - `test_webhook.py`: test_message_activity_forwarded, test_invoke_activity_forwarded, test_channel_mention_forwarded, test_channel_non_mention_ignored, test_attachment_rejected
       - `test_send.py`: test_send_text_activity, test_send_adaptive_card, test_threaded_reply
       - Mock httpx calls, mock token validation (always valid in tests)
