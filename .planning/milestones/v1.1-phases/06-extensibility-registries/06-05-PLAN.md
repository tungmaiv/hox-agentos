    **Safe Expression Evaluator (backend/skills/safe_eval.py):**

    ```python
    import ast

    def safe_eval_condition(expression: str, variables: dict[str, Any]) -> bool:
        """Evaluate a condition expression safely using AST parsing.

        Supports ONLY:
        - Comparisons: X > N, X < N, X >= N, X <= N, X == Y, X != Y
        - Function calls: len(X)
        - Variables: {{step_id.output}} already resolved to values
        - Literals: integers, strings, booleans, None

        NO eval(), NO exec(), NO arbitrary code execution.
        """
    ```

    Implementation:
    - Parse expression with `ast.parse(expression, mode='eval')`
    - Walk the AST tree with a custom `SafeEvaluator(ast.NodeVisitor)`:
      - `ast.Compare`: evaluate left and comparators with operators
      - `ast.Call`: only allow `len()` function, reject all others
      - `ast.Name`: look up in variables dict
      - `ast.Constant`: return value directly (int, str, bool, None)
      - `ast.BoolOp`: support `and`, `or`
      - Reject all other node types (raise `UnsafeExpressionError`)
    - Before parsing, resolve `{{step_id.output}}` templates in expression string to Python-safe variable names (e.g., replace `{{fetch.output}}` with `_var_fetch_output`)
    - Build variables dict mapping `_var_fetch_output` -> actual value
    - Raise `UnsafeExpressionError` for: attribute access, subscript, lambda, import, exec, eval, comprehensions

    **SkillValidator (backend/skills/validator.py):**

    ```python
    class SkillValidator:
        def validate_procedure(self, procedure_json: dict) -> list[str]:
            """Returns list of error messages. Empty = valid."""
    ```

    Validation rules:
    - `schema_version` must be present and equal to "1.0"
    - `steps` must be a non-empty list
    - `steps` length <= 20
    - Each step must have `id` (unique across steps) and `type` (one of: tool, llm, condition)
    - `tool` steps require `tool` field (string)
    - `llm` steps require `model_alias` and `prompt_template` fields
    - `condition` steps require `expression` and `true_step` fields
    - `output` template must reference existing step IDs via `{{step_id.output}}`
    - Variable references `{{step_id.output}}` in prompt_template/params must reference prior step IDs
    - No circular references (condition true_step/false_step must not create cycles)
    - Prompt template size <= 10KB per step
    - Condition expressions must be parseable by `safe_eval_condition` (dry-run parse)

    **SkillExecutor (backend/skills/executor.py):**

    ```python
    @dataclass
    class StepContext:
        user_input: dict | None
        outputs: dict[str, Any]

    @dataclass
    class SkillResult:
        success: bool
        output: str
        step_outputs: dict[str, Any]
        failed_step: str | None = None

    class SkillExecutor:
        async def run(self, skill: SkillDefinition, user_context: UserContext, session: AsyncSession, user_input: dict | None = None, event_emitter: Callable | None = None) -> SkillResult:
    ```

    Implementation:
    - Iterate through `procedure_json["steps"]` sequentially
    - **tool step**: Call `get_tool(tool_name, session)` to verify tool exists and is active. Call `check_tool_acl(user_context["user_id"], tool_name, session)` for Gate 3. If denied, raise SkillStepError. Resolve params using `_resolve_template()`. Call the tool handler function (import via handler_module/handler_function from tool_definitions).
    - **llm step**: Call `get_llm(step["model_alias"])` to get ChatOpenAI instance. Resolve `prompt_template` variables. Invoke LLM. Store response as step output.
    - **condition step**: Call `safe_eval_condition(expression, context.outputs)` from skills/safe_eval.py. **NO eval() -- AST-based only.** Route to `true_step` or `false_step` (skip steps until target). For MVP: conditions only skip forward, no loops.
    - **Variable resolution** `_resolve_template(template, context)`: Replace `{{step_id.output}}` with `str(context.outputs[step_id])`. Simple string replacement, no Jinja2.
    - **Error handling**: Wrap each step in try/except. On failure, return SkillResult(success=False, output=error_message, failed_step=step["id"]).
    - **AG-UI streaming**: If `event_emitter` callback is provided, emit step-progress events after each step completes:
      ```python
      if event_emitter:
          await event_emitter({
              "type": "step_progress",
              "skill_name": skill.name,
              "step_id": step["id"],
              "step_type": step["type"],
              "status": "completed",  # or "failed"
              "description": step.get("description", ""),
          })
      ```
      The event_emitter is wired to AG-UI EventEncoder in the skill_executor graph node (plan 06-06).
    - **Audit logging**: Log each step execution via `structlog.get_logger(__name__)` with skill_name, step_id, step_type, tool_name (if tool step), user_id, duration_ms.

    **SecurityScanner (backend/skills/security_scanner.py):**

    ```python
    INJECTION_PATTERNS = [
        r"ignore\s+(all\s+)?previous\s+instructions",
        r"forget\s+(everything|all|your)",
        r"you\s+are\s+now\s+a",
        r"system\s*:\s*",
        r"<\|im_start\|>",
        r"Human:\s*|Assistant:\s*",
        r"(curl|wget|fetch)\s+http",
        r"base64\.(encode|decode)",
        r"eval\(|exec\(|__import__",
    ]

    @dataclass
    class SecurityReport:
        score: int                    # 0-100
        factors: dict[str, int]       # factor_name -> score
        recommendation: str           # "approve", "review", "reject"
        injection_matches: list[str]  # matched patterns

    class SecurityScanner:
        def scan(self, skill_data: dict, source_url: str | None = None) -> SecurityReport:
    ```

    Scoring with weights from design doc Section 9.4:
    - **Source reputation (30%)**: Check source_url domain against trusted_sources config. Known = high score, unknown URL = 20, manual paste = 40.
    - **Tool scope (25%)**: Analyze referenced tools. Read-only = 100, write = 60, sandbox = 30, admin = 0. Average across all tool steps.
    - **Prompt safety (25%)**: Scan instruction_markdown and all prompt_template fields against INJECTION_PATTERNS. No matches = 100, each match -50 (floor 0).
    - **Complexity (10%)**: Step count. <=5 = 100, 6-10 = 70, 11-20 = 40.
    - **Author verification (10%)**: Signed = 100, unsigned = 50 (MVP: always unsigned = 50).

    Recommendation: score >= 80 -> "approve", score >= 60 -> "review", score < 60 -> "reject".
