## Decisions Made

- **fetch_user_realm_roles uses `get_settings()` (lazy) not module-level `settings`** -- avoids import-time side effects in tests; same pattern as jwt.py
- **format_for_channel extracted as module-level function with instance method wrappers** -- enables import from node_handlers without ChannelGateway instance; preserves backward compat for _invoke_agent
- **Sub-agent imports are lazy (inside _handle_agent_node body)** -- avoids circular deps when node_handlers is loaded before subagents; tests patch at definition site per CLAUDE.md gotcha
- **Keycloak failure immediately fails workflow run** -- security-first per locked decision; no fallback to stale owner_roles_json
- **resolved_roles added to user_context** -- audit trail: workflow run metadata shows actual Keycloak roles used for ACL

## User Setup Required

**Keycloak Admin Setup:** The `blitz-backend` service account must have the `realm-management` > `view-users` client role assigned in Keycloak. Without this, the `fetch_user_realm_roles` call returns 403, and workflows will fail with "Keycloak role fetch failed". This is a Keycloak admin console configuration step, not a code change.
