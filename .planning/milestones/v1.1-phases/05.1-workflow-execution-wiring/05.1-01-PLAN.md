New module with one async function: `fetch_user_realm_roles(user_id: str) -> list[str]`.

Implementation:
- Use `httpx.AsyncClient` with `verify=settings.keycloak_ca_cert or True` and `timeout=10.0`
- Step 1: POST to `{keycloak_url}/realms/{keycloak_realm}/protocol/openid-connect/token` with `grant_type=client_credentials`, `client_id=settings.keycloak_client_id`, `client_secret=settings.keycloak_client_secret`
- Step 2: GET `{keycloak_url}/admin/realms/{keycloak_realm}/users/{user_id}/role-mappings/realm` with `Authorization: Bearer {admin_token}`
- Parse response JSON: extract `[r["name"] for r in response.json()]`
- Use `structlog.get_logger(__name__)` for logging
- Log the fetched role list: `logger.info("keycloak_roles_fetched", user_id=user_id, roles=roles)`
- On any `httpx.HTTPError`: let it propagate (caller handles per locked decision: "if Keycloak unreachable, fail workflow run")
- Import `get_settings` from `core.config` (not `settings` module-level) to avoid import-time side effects in tests

  - Get `owner_user_id` from `user_context.get("user_id")`
  - If no `owner_user_id`, raise `ValueError("No user_id in workflow context")` (caught by executor, sets run failed)
  - Query `channel_accounts` table:
    ```python
    from core.models.channel import ChannelAccount
    from sqlalchemy import and_, select
    uid = uuid.UUID(str(owner_user_id))
    async with async_session() as session:
        result = await session.execute(
            select(ChannelAccount).where(
                and_(
                    ChannelAccount.user_id == uid,
                    ChannelAccount.channel == channel,
                    ChannelAccount.is_paired == True,
                )
            )
        )
        account = result.scalar_one_or_none()
    ```
  - If no account found: raise `ValueError(f"No linked {channel} account for user {owner_user_id}")` per locked decision
  - Build `InternalMessage` with `external_user_id=account.external_user_id` AND `external_chat_id=account.external_user_id` (for DMs, chat_id == user_id per Pitfall 1)
  - Call `gateway.send_outbound(msg)` (existing 3x backoff)
  - Log delivery metadata: `logger.info("channel_output_delivered", channel=channel, user_id=str(owner_user_id), external_chat_id=account.external_user_id)`
  - Return `{"channel": channel, "message": message, "sent": True}`
- On delivery failure (send_outbound raises), let the exception propagate — the workflow executor catches it and marks the run as failed with node-level error
