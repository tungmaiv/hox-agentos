### Integration Warning: Builder Metadata Fields

**Affected requirements:** SKBLD-01/02/03 (builder output) + SKSTD-02 (metadata stored and visible)

The artifact builder LangGraph's `fill_form` tool mapping and prompts do not include `allowed_tools`, `category`, or `tags` fields. Skills created through the **builder chat flow** will always have these three fields as `null` in the DB.

**Impact:**
- Builder-created skills are invisible to `?category=X` filter in catalog
- `SkillMetadataPanel` shows no metadata chips for builder-created skills
- Skill executor pre-gate (`SKSEC-02`) passes (null allowed_tools = unrestricted, which is the default)

**Not affected:**
- ZIP import, one-click repo import, SKILL.md import — all correctly populate these fields
- Export of builder-created skills produces valid ZIP (fields are null but valid)
- Security scanning still runs (trust score calculated without tool-permission factor)

**Fix location:**
- `backend/agents/artifact_builder_prompts.py` — add allowed_tools, category, tags guidance
- `backend/agents/artifact_builder.py` — add form field mappings in `_DRAFT_TO_FORM_FIELDS`
