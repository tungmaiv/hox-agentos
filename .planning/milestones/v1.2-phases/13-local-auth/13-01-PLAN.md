**1. Refactor `backend/security/jwt.py` — dual-issuer dispatch:**

Refactor `validate_token()` to peek at the `iss` claim first (unverified), then dispatch:
- `iss == settings.keycloak_issuer` → `_validate_keycloak_token(token)` (existing RS256 + JWKS logic, extracted from current validate_token body — zero behavior change)
- `iss == "blitz-local"` → import and call `validate_local_token(token)` from `security.local_auth`
- Anything else → `HTTPException(401, "Unknown token issuer")`

Use `jose_jwt.get_unverified_claims(token)` to peek at issuer without signature verification. The existing JWKS fetch + RS256 decode becomes the body of `_validate_keycloak_token()`. All existing tests must continue to pass because the Keycloak path is unchanged.
