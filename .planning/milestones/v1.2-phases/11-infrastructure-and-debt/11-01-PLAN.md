  <name>Task 1: Create PromptLoader in backend/core/prompts.py</name>
  <files>backend/core/prompts.py</files>
  <action>
Create `backend/core/prompts.py` with a module-level `load_prompt(name: str, **vars: str) -> str` function that:

1. Resolves the prompt file path as: `Path(__file__).parent.parent / "prompts" / f"{name}.md"` — this works from any working directory since it is relative to the source file.

2. Uses a module-level dict `_cache: dict[str, str] = {}` for in-memory caching. Cache key is the prompt name (not the rendered string — cache the raw template, render on each call with the provided vars).

3. On cache miss: read the file with `Path.read_text(encoding="utf-8")`. On cache hit: use cached raw template. Raise `FileNotFoundError` with a clear message if the .md file does not exist (do not silently return empty string).

4. Performs variable substitution using Jinja2-style `{{ var_name }}` placeholders (double braces). Implementation: use `str.replace("{{ " + key + " }}", value)` for each key/value pair in vars — no Jinja2 library needed, this is the simplest approach that covers all use cases. Order of substitution does not matter since placeholders are unique per prompt.

5. Dev vs prod caching: check `os.getenv("ENVIRONMENT", "production")`. In `"development"` mode, bypass the cache (always re-read disk) so prompt edits are visible without restart. In all other modes (production, test), use the cache.

6. Add a `clear_cache() -> None` function for test isolation.

Full type annotations required. Use `structlog.get_logger(__name__)` for a debug log on cache miss. No bare `dict`, no `Any`.

Example usage that must work:
```python
from core.prompts import load_prompt

# No vars
system = load_prompt("master_agent")

# With vars
prompt = load_prompt("intent_classifier", message="check my emails")
```
