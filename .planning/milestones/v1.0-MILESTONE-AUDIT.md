### Item 8 Detail (MEMO-02 role-label bug)

**File:** `backend/scheduler/tasks/embedding.py`, line 163

```python
# Current (wrong) — condition never true; every turn labeled "Assistant"
f"{'User' if t.role == 'human' else 'Assistant'}: {t.content}"

# Correct fix
f"{'User' if t.role == 'user' else 'Assistant'}: {t.content}"
```

`ConversationTurn.role` stores `"user"` (not `"human"`). The condition `t.role == 'human'` is always False. Every turn — user and assistant alike — is formatted as `"Assistant: ..."` in the summarization transcript. The `blitz/summarizer` LLM receives a malformed transcript, producing episode summaries with incorrect speaker attribution. Episodes are created, stored, and injected correctly; only quality is degraded.
